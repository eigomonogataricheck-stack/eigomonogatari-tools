5. 編集ガイド

キャラを追加・変更したい

→ 図鑑スプレッドシートの 「図鑑マスター」シート を編集するだけ。





行を追加：大分類〜画像URLの6列を埋める



行を削除：その行を消す



分類変更：A〜D列を書き換え



コード変更不要。即反映。

Tier計算ロジックを変えたい

→ GASの Code.gs を編集。





generateTierAdvanced() は別途定義されている関数（Code.gs内またはライブラリ）



変更後は必ず 再デプロイ が必要（後述）

見た目・UIを変えたい

→ GitHubの index.html を編集 → Commit。





GAS内の index.html は GAS版UI用（script.google.com直接アクセス用）



GitHub Pages版が公開サイト



6. デプロイ手順

GAS（Code.gs を変更した場合）





スクリプトエディタで保存



「デプロイ」→「デプロイを管理」



鉛筆アイコン（✏️）



バージョンを「新しいバージョン」に変更



「デプロイ」





⚠️「新しいデプロイ」を選ぶとURLが変わるので使わない



「デプロイを管理」から更新すればURLは同じまま

GitHub Pages（index.html を変更した場合）





https://github.com/eigomonogataricheck-stack/eigomonogatari-tools で index.html をクリック



鉛筆アイコン（✏️）→ 編集



「Commit changes」



1〜2分後に自動反映



7. API仕様

GitHub Pages の JS から GAS を呼ぶ。全て GET リクエスト。







エンドポイント



説明



レスポンス





?api=master



選択肢データ取得



{shibari:[], rank:[], country:[], targetChars:[]}





?api=zukan



図鑑全データ取得



[{d, c, s, h, n, u}, ...]





?api=tier&mode=...&attribute=...



Tier生成・結果取得



{success:true, data:[{rankName, images:[url,...]}]}





パラメータなし



GAS版HTMLを返す



HTML

図鑑データのフィールド





d: 大分類（通常ゆる / 特殊ゆる）



c: 中分類（国内, 対戦, etc.）



s: 小分類（関東, 国外, etc.）



h: 見出し（東京, ベトナム, etc.）



n: キャラ名



u: 画像URL



8. 制限事項・注意点

9. 図鑑マスター再生成

「ゆる図鑑」シートから「図鑑マスター」を再生成する関数が Code.gs に存在する。





関数名: createZukanMaster()



スクリプトエディタで手動実行



実行すると図鑑マスターシートを削除→再作成



⚠️ 手動修正した図鑑マスターの内容は上書きされるので注意



10. トラブルシューティング







症状



原因・対処





GitHub Pages が更新されない



Commit後1〜2分待つ。キャッシュクリア（Ctrl+Shift+R）





GAS変更が反映されない



「デプロイを管理」→ 新しいバージョンでデプロイしたか確認





図鑑が0体



図鑑マスターシートが空 or シート名が違う





Tier生成でエラー



tier条件入力シートの構成変更、またはgenerateTierAdvanced関数の問題





API通信失敗



GASのデプロイURLが正しいか、アクセス権が「全員」になっているか確認
